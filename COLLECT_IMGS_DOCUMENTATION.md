# 📄 เอกสารฟังก์ชัน - collect_imgs.py

ไฟล์นี้ใช้สำหรับเก็บรวบรวมข้อมูลภาพตัวอักษร A-Z สำหรับการฝึกสอนโมเดล hand gesture detection

---

## 📋 รายการฟังก์ชั่นทั้งหมด

### 1. `adjust_brightness_contrast(frame, brightness=0, contrast=50)`
**ประเภท:** ฟังก์ชันสำเร็จรูป (Signal Processing Function)

**วัตถุประสงค์:** ปรับความสว่างและ contrast ของภาพ

**พารามิเตอร์:**
- `frame` - ภาพ OpenCV
- `brightness` - ค่าความสว่าง (-100 ถึง 100)
  - ค่าบวก = สว่างขึ้น
  - ค่าลบ = มืดลง
  - ค่าเริ่มต้น: 0 (ไม่เปลี่ยน)
- `contrast` - ค่า contrast (0 ถึง 100)
  - ค่าเริ่มต้น: 50 (ปกติ)
  - 0 = เล็กจิตร
  - 100 = คมชัดสูง

**ค่าที่ส่งกลับ:** ภาพที่ปรับแล้ว

**วิธีการ:**
- คำนวณค่า alpha และ beta สำหรับความสว่าง
- ใช้ cv2.convertScaleAbs เพื่อปรับแต่ละช่อง
- ตัดค่าออกเพื่อให้อยู่ในช่วง 0-255

---

### 2. `enhance_image_visibility(frame)`
**ประเภท:** ฟังก์ชันปรับปรุงภาพ (Image Enhancement Function)

**วัตถุประสงค์:** เพิ่มความชัดของภาพโดยใช้ CLAHE (Contrast Limited Adaptive Histogram Equalization)

**พารามิเตอร์:**
- `frame` - ภาพ OpenCV (BGR format)

**ขั้นตอนการทำงาน:**
1. แปลงภาพจาก BGR เป็น LAB color space
2. แยก L channel (ความสว่าง)
3. ใช้ CLAHE บน L channel
   - clipLimit: 3.0 (จำกัดอาหาร)
   - tileGridSize: (8, 8) (ขนาดกระเบื้อง)
4. รวม L, a, b channels กลับ
5. แปลงกลับเป็น BGR

**ค่าที่ส่งกลับ:** ภาพที่ปรับปรุงแล้ว

**ข้อดี:** ทำให้รูปชัดขึ้นโดยไม่มีเสียงรบกวนมากเกินไป

---

### 3. `draw_countdown(frame, countdown_value, text="กำลังจับภาพใน...")`
**ประเภท:** ฟังก์ชันวาดภาพ (Drawing Function)

**วัตถุประสงค์:** วาด countdown timer บนภาพ

**พารามิเตอร์:**
- `frame` - ภาพ OpenCV
- `countdown_value` - ตัวเลข countdown (1-3)
- `text` - ข้อความแสดง (ค่าเริ่มต้น: "กำลังจับภาพใน...")

**องค์ประกอบที่วาด:**
1. **พื้นหลัง:** สี่เหลี่ยมใหญ่โปร่งแสง (สีดำ 50%)
2. **ข้อความ:** ข้อความแจ้งสีเขียว
3. **ตัวเลข:** เลขนับถอยหลังใหญ่มาก สีเหลืองเขียว

**ค่าที่ส่งกลับ:** ภาพที่วาด countdown แล้ว

---

### 4. `draw_collection_status(frame, current_count, total_count, elapsed_time)`
**ประเภท:** ฟังก์ชันวาดภาพ (Drawing Function)

**วัตถุประสงค์:** แสดงสถานะการเก็บภาพ (กำลังจับภาพกี่รูป สัดส่วน เวลา)

**พารามิเตอร์:**
- `frame` - ภาพ OpenCV
- `current_count` - จำนวนรูปที่จับแล้ว
- `total_count` - จำนวนรูปทั้งหมดที่ต้องจับ
- `elapsed_time` - เวลาที่ผ่านไป (วินาที)

**องค์ประกอบที่วาด:**
1. **ข้อมูลบน:** "📸 จับภาพ: X/Y" (สีเขียว)
2. **เวลา:** "⏱️ เวลา: X.Xs" (สีเหลืองเขียว)
3. **Progress bar:** แถบความก้าวหน้า
   - พื้นหลัง: สีเทา
   - Filled: สีเขียว (ตามสัดส่วน)
   - เปอร์เซ็นต์: ข้อความสีขาว

**ค่าที่ส่งกลับ:** ภาพที่เพิ่มข้อมูลสถานะแล้ว

---

### 5. `draw_controls_info(frame)`
**ประเภท:** ฟังก์ชันวาดภาพ (Drawing Function)

**วัตถุประสงค์:** แสดงปุ่มควบคุมที่ด้านล่างของภาพ

**พารามิเตอร์:**
- `frame` - ภาพ OpenCV

**ข้อมูลที่แสดง:**
```
ปุ่มควบคุม: Q=ออก | B/V=ความสว่าง | C/X=Contrast
         SPACE=Toggle กล้อง
```

**สี:** สีเหลืองอมส้ม

**ค่าที่ส่งกลับ:** ภาพที่เพิ่มข้อมูลปุ่มแล้ว

---

### 6. `wait_for_ready(cap, letter)`
**ประเภท:** ฟังก์ชันรอการยืนยัน (Confirmation Function)

**วัตถุประสงค์:** รอให้ผู้ใช้พร้อมจับภาพตัวอักษรที่ระบุ อนุญาตให้ปรับความสว่างและ contrast

**พารามิเตอร์:**
- `cap` - VideoCapture object
- `letter` - ตัวอักษรที่จะจับ (A-Z)

**ขั้นตอนการทำงาน:**
1. แสดง "พร้อมจับตัวอักษร X ใช่ไหม?"
2. รอให้ผู้ใช้กด Q เพื่อยืนยัน
3. อนุญาตให้ปรับการตั้งค่า:
   - B/V: ความสว่าง
   - C/X: Contrast
   - SPACE: เปิด/ปิดกล้อง
4. เมื่อกด Q -> ออกจากฟังก์ชันเพื่อดำเนินการจับภาพ

**ปุ่มควบคุม:**
- `Q` - เริ่มจับภาพ
- `B` - เพิ่มความสว่าง (+5)
- `V` - ลดความสว่าง (-5)
- `C` - เพิ่ม Contrast (+5)
- `X` - ลด Contrast (-5)
- `SPACE` - เปิด/ปิดกล้อง

---

### 7. `collect_samples(cap, class_dir, letter)`
**ประเภท:** ฟังก์ชันหลัก (Main Function)

**วัตถุประสงค์:** จับภาพตัวอักษรหนึ่งตัว DATASET_SIZE รูป พร้อมนับถอยหลังและความล่าช้า

**พารามิเตอร์:**
- `cap` - VideoCapture object
- `class_dir` - โฟลเดอร์ที่จะบันทึกรูป
- `letter` - ตัวอักษรที่จับ

**ขั้นตอนการทำงาน:**
1. **ลูปจั่ว DATASET_SIZE รูป**
   
   **สำหรับแต่ละรูป:**
   - **นับถอยหลัง:** (COUNTDOWN_SECONDS: 3 วินาที)
     - แสดงหน้าจอ countdown
     - อนุญาตให้ปรับการตั้งค่า
   
   - **จับภาพจริง:**
     - อ่านเฟรมจากกล้อง
     - ปรับปรุงภาพ (CLAHE)
     - ปรับความสว่าง/contrast
     - บันทึกเป็น `{number}.jpg`
     - แสดง "SAVED! ✓"

2. **เมื่อจับครบ DATASET_SIZE:** พิมพ์ข้อมูลสรุป

**ปุ่มควบคุม:**
- `Q` - หยุดจับและข้ามไปตัวอักษรถัดไป
- `B/V/C/X/SPACE` - ปรับการตั้งค่า

**ค่าที่ส่งกลับ:** 
- `True` - จับสำเร็จหรือผู้ใช้ยุติ
- `False` - มีข้อผิดพลาด

---

### 8. `main()`
**ประเภท:** ฟังก์ชันหลัก (Main Entry Point)

**วัตถุประสงค์:** จุดเข้าหลักของโปรแกรม - จับภาพตัวอักษร A-Z ทั้งหมด

**ขั้นตอนการทำงาน:**
1. **สร้างโฟลเดอร์ `./data`** หากยังไม่มี
2. **สำหรับแต่ละตัวอักษร j = 0 ถึง 25 (A-Z):**
   - สร้างโฟลเดอร์สำหรับตัวอักษร (`./data/0`, `./data/1`, ...)
   - เปิดกล้อง (ความละเอียด 1280x720, FPS 30)
   - เรียก `wait_for_ready(cap, letter)` - รอให้พร้อม
   - เรียก `collect_samples(cap, class_dir, letter)` - จับภาพ
   - ถ้าเกิดข้อผิดพลาด -> ออก
3. **เมื่อจับครบทั้ง 26 ตัว:**
   - ปิดกล้อง
   - ปิด window
   - พิมพ์สรุป

**ผลลัพธ์:** 
```
./data/
  ├── 0/  (ตัวอักษร A - 100 รูป)
  ├── 1/  (ตัวอักษร B - 100 รูป)
  ├── ...
  └── 25/ (ตัวอักษร Z - 100 รูป)
```

## 📊 กระบวนการจับภาพ

```
เริ่มต้น
   ↓
สำหรับแต่ละตัวอักษร (A-Z)
   ↓
wait_for_ready() - รอให้ผู้ใช้พร้อม
   ↓
collect_samples() - จับภาพ DATASET_SIZE รูป
   ├─ นับถอยหลัง 3 วินาที
   ├─ จับภาพ 1 รูป
   ├─ บันทึกรูป
   ├─ แสดง "SAVED! ✓"
   └─ ซ้ำจนครบ DATASET_SIZE
   ↓
รูป DATASET_SIZE × NUMBER_OF_CLASSES = 2,600 รูป
   ↓
พิมพ์สรุป
   ↓
จบ
```

---

## ⏱️ เวลาที่คาดหวัง

- **เวลาต่อตัวอักษร:** ประมาณ 5-10 นาที
  - นับถอยหลัง: 3 วินาที × 100 รูป = 5 นาที
  - ความล่าช้าระหว่างรูป: ~1-5 วินาที × 100 = 1-5 นาที

- **เวลารวมทั้งหมด:** ประมาณ 2-3 ชั่วโมง สำหรับ 26 ตัวอักษร

## 🔧 การปรับแต่ง

หากต้องการเปลี่ยนค่าต่างๆ ให้แก้ไขตัวแปรที่ด้านบนของไฟล์:

```python
DATA_DIR = './data'              # เปลี่ยนโฟลเดอร์เก็บข้อมูล
NUMBER_OF_CLASSES = 26           # เปลี่ยนจำนวนตัวอักษร

```

## 📝 หมายเหตุ

- ภาพจะปรับปรุงด้วย CLAHE และความสว่าง/contrast ก่อนบันทึก
- คุณสามารถปรับความสว่างขณะจับภาพเพื่อให้ได้ผลลัพธ์ที่ดีที่สุด
- ข้อมูลที่เก็บจะใช้สำหรับฝึกสอนโมเดล hand gesture classification
- กระดาษข้อมูลที่ดีจะช่วยให้โมเดลมีความแม่นยำสูงขึ้น
